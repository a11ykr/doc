WCAG 2.2의 기법

요약

기법은 WCAG(웹 콘텐츠 접근성 지침)를 준수하는 방법의 예이다. WCAG 준수에 필수적이지는 않다. 자세한 내용은 WCAG 기술 정보를 참조하라.

ARIA 기법§

  • ARIA1: aria-describedby 속성을 사용하여 사용자 인터페이스 컨트롤에 대한 설명 레이블 제공
  • ARIA2: aria 필수 속성으로 필수 필드 식별
  • ARIA4: WAI-ARIA 역할을 사용하여 사용자 인터페이스 구성 요소의 역할 노출
  • ARIA5: WAI-ARIA 상태 및 속성 속성을 사용하여 사용자 인터페이스 구성 요소의 상태 노출
  • ARIA6: aria-label을 사용하여 객체에 대한 레이블 제공
  • ARIA7: 링크 목적으로 aria-labelledby 사용
  • ARIA8: 링크 목적으로 aria-label 사용
  • ARIA9: aria-labelledby를 사용하여 여러 텍스트 노드의 레이블 연결
  • ARIA10: aria-labelledby를 사용하여 텍스트가 아닌 콘텐츠에 대한 텍스트 대안 제공
  • ARIA11: ARIA 랜드마크를 사용하여 페이지 영역 식별
  • ARIA12: 역할=제목을 사용하여 제목 식별
  • ARIA13: aria-labelledby를 사용하여 지역 및 랜드마크 이름 지정
  • ARIA14: aria-label을 사용하여 보이는 라벨을 사용할 수 없는 곳에 보이지 않는 라벨 제공
  • ARIA15: aria-describedby를 사용하여 이미지 설명 제공
  • ARIA16: aria-labelledby를 사용하여 사용자 인터페이스 컨트롤의 이름 제공
  • ARIA17: 그룹화 역할을 사용하여 관련 양식 컨트롤 식별
  • ARIA18: aria-alertdialog를 사용하여 오류 식별
  • ARIA19: ARIA role=alert 또는 라이브 영역을 사용하여 오류 식별
  • ARIA20: 지역 역할을 사용하여 페이지 영역 식별
  • ARIA21: aria-invalid를 사용하여 오류 필드 표시
  • ARIA22: role=status를 사용하여 상태 메시지 표시
  • ARIA23: role=log를 사용하여 순차적 정보 업데이트 식별
  • ARIA24: role="img"로 글꼴 아이콘을 의미론적으로 식별

클라이언트측 스크립트 기법§

  • SCR1: 사용자가 기본 시간 제한을 연장하도록 허용
  • SCR2: 중복 키보드 및 마우스 이벤트 핸들러 사용
  • SCR14: 스크립트를 사용하여 중요하지 않은 경고를 선택 사항으로 만들기
  • SCR16: 사용자에게 시간 제한이 곧 만료됨을 경고하는 스크립트 제공
  • SCR18: 클라이언트측 검증 및 경고 제공
  • SCR19: 컨텍스트 변경을 유발하지 않고 선택 요소에서 onchange 이벤트 사용
  • SCR20: 키보드 및 기타 장치별 기능을 모두 사용
  • SCR22: 스크립트를 사용하여 깜박임을 제어하고 5초 이내에 중지합니다.
  • SCR24: 점진적인 향상을 사용하여 사용자 요청 시 새 창 열기
  • SCR26: 트리거 요소 바로 다음에 문서 개체 모델에 동적 콘텐츠 삽입
  • SCR27: 문서 개체 모델을 사용하여 페이지 섹션 재정렬
  • SCR28: 확장 및 축소 가능한 메뉴를 사용하여 콘텐츠 차단 우회
  • SCR29: 정적 HTML 요소에 키보드 액세스 가능 작업 추가
  • SCR30: 스크립트를 사용하여 링크 텍스트 변경
  • SCR31: 스크립트를 사용하여 포커스가 있는 요소의 배경색이나 테두리를 변경합니다.
  • SCR32: 클라이언트 측 유효성 검사 제공 및 DOM을 통해 오류 텍스트 추가
  • SCR33: 스크립트를 사용하여 콘텐츠를 스크롤하고 콘텐츠를 일시 중지하는 메커니즘 제공
  • SCR34: 텍스트 크기에 따라 크기와 위치를 계산합니다.
  • SCR35: 앵커 및 버튼의 onclick 이벤트를 사용하여 작업 키보드에 액세스할 수 있도록 만들기
  • SCR36: 사용자가 정적 창이나 영역에서 텍스트 이동, 스크롤 또는 자동 업데이트를 표시할 수 있는 메커니즘 제공
  • SCR38: 점진적인 향상으로 디자인된 웹 페이지에 적합한 대체 버전 만들기
  • SCR39: 포커스나 호버에 있는 콘텐츠를 가리킬 수 있고, 무시할 수 있고, 지속적으로 만들기

일반적인 실패§

  • F1: CSS를 사용하여 정보 위치를 지정하여 콘텐츠의 의미를 변경하여 성공 기준 1.3.2 실패
  • F2: 적절한 마크업이나 텍스트를 사용하지 않고 정보를 전달하기 위해 텍스트 표현의 변경 사항을 사용하여 성공 기준 1.3.1 실패
  • F3: 중요한 정보를 전달하는 이미지를 포함하기 위해 CSS를 사용하여 적합 기준 1.1.1 실패
  • F4: 5초 이내에 중지할 수 있는 메커니즘 없이 text-꾸밈:점멸을 사용하여 성공 기준 2.2.2 실패
  • F7: 5초 이상 객체 또는 애플릿으로 인해 적합 기준 2.2.2 실패
  • F8: 캡션에서 일부 대화나 중요한 음향 효과가 누락되어 적합 기준 1.2.2에 부적합
  • F9: 사용자가 양식 요소에서 포커스를 제거할 때 컨텍스트 변경으로 인해 적합 기준 3.2.5 실패
  • F10: 사용자를 하나의 형식 유형 안에 가두는 방식으로 여러 콘텐츠 형식을 결합하여 성공 기준 2.1.2 및 적합성 요구 사항 5에 부적합
  • F12: 메커니즘 재인증 없이 세션 시간 제한이 있어서 적합 기준 2.2.5 실패
  • F13: 이미지의 색상 차이로 전달되는 정보를 포함하지 않는 텍스트 대안이 있어 적합 기준 1.1.1 및 1.4.1에 부적합
  • F14: 모양이나 위치로만 콘텐츠를 식별하여 적합 기준 1.3.3에 부적합
  • F15: 기술에 대한 접근성 API를 사용하지 않거나 불완전하게 사용하는 사용자 정의 컨트롤 구현으로 인해 적합 기준 4.1.2 실패
  • F16: 콘텐츠를 일시 중지하고 다시 시작하는 메커니즘을 포함하지 않고 활동에 움직임이 필수적이지 않은 스크롤 콘텐츠를 포함하여 성공 기준 2.2.2 실패
  • F19: 사용자가 부적합 웹 페이지의 대체 적합 버전을 찾을 수 있는 방법을 제공하지 않아 적합성 요구 사항 1의 실패
  • F20: 텍스트가 아닌 콘텐츠가 변경될 때 텍스트 대체를 업데이트하지 않아 적합 기준 1.1.1 및 4.1.2에 부적합
  • F22: 사용자가 요청하지 않은 창 열기로 인해 적합 기준 3.2.5 실패
  • F23: 소리를 끌 수 있는 메커니즘이 없는 3초 이상의 소리 재생으로 인해 1.4.2 오류가 발생했습니다.
  • F24: 배경색을 지정하지 않고 전경색을 지정하거나 그 반대의 경우로 인해 적합 기준 1.4.3, 1.4.6 및 1.4.8에 부적합
  • F25: 콘텐츠를 식별하지 않는 웹 페이지 제목으로 인해 적합 기준 2.4.2에 부적합
  • F26: 정보 전달을 위해 그래픽 기호만 사용하여 적합 기준 1.3.3에 부적합
  • F30: 대체가 아닌 텍스트 대체(예: 파일 이름 또는 자리 표시자 텍스트) 사용으로 인해 적합 기준 1.1.1 및 1.2.1에 부적합
  • F31: 웹 페이지 집합 내의 서로 다른 웹 페이지에서 동일한 기능에 대해 서로 다른 두 개의 레이블을 사용하여 성공 기준 3.2.4에 부적합
  • F32: 단어 내 간격을 제어하기 위해 공백 문자를 사용하여 성공 기준 1.3.2에 부적합
  • F33: 공백 문자를 사용하여 일반 텍스트 콘텐츠에 여러 열을 생성하여 적합 기준 1.3.1 및 1.3.2에 부적합
  • F34: 일반 텍스트 콘텐츠의 테이블 형식을 지정하기 위해 공백 문자를 사용하여 적합 기준 1.3.1 및 1.3.2에 부적합
  • F36: 양식 자동 제출 및 값 제공으로 인해 적합 기준 3.2.2 실패
  • F37: 라디오 버튼, 확인란 또는 선택 목록의 선택이 변경될 때 사전 경고 없이 새 창을 실행하여 적합 기준 3.2.2 실패
  • F38: 보조 기술이 무시할 수 있는 방식으로 HTML의 장식 이미지를 마크업하지 않아 적합 기준 1.1.1에 부적합
  • F39: 보조 기술에서 무시해야 하는 이미지에 대해 null이 아닌 텍스트 대체(예: alt="spacer" 또는 alt="image") 제공으로 인해 적합 기준 1.1.1에 부적합
  • F40: 시간 제한이 있는 메타 리디렉션 사용으로 인한 실패
  • F41: 페이지를 다시 로드하기 위해 메타 새로 고침을 사용하여 성공 기준 2.2.1, 2.2.4 및 3.2.5 실패
  • F42: 링크를 에뮬레이션할 때 성공 기준 1.3.1, 2.1.1, 2.1.3 또는 4.1.2 실패
  • F43: 콘텐츠의 관계를 나타내지 않는 방식으로 구조적 마크업을 사용하여 적합 기준 1.3.1에 부적합
  • F44: 의미와 조작성을 보존하지 않는 탭 순서를 생성하기 위해 tabindex를 사용하여 성공 기준 2.4.3 실패
  • F46: 요소, 레이아웃 테이블 사용으로 인해 적합 기준 1.3.1 실패
  • F47: 깜박임 요소 사용으로 인해 적합 기준 2.2.2 실패
  • F48: 표 형식 정보를 마크업하기 위해 사전 요소를 사용하여 적합 기준 1.3.1 실패
  • F49: 선형화 시 의미가 없는 HTML 레이아웃 테이블을 사용하여 적합 기준 1.3.2에 부적합
  • F50: 5초 이하로 깜박임을 멈추는 메커니즘 없이 깜박임 효과를 발생시키는 스크립트로 인해 적합 기준 2.2.2에 부적합함
