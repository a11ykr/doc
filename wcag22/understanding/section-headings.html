성공기준 이해 2.4.10:Section Headings (Level AAA)

성공 기준 (SC)

콘텐츠는 섹션 제목을 사용하여 구조화해야 한다.

참고

“제목”(Heading)은 일반적인 의미로 사용되며, 타이틀(titles)과 상이한 유형의 콘텐츠에 제목을 추가하는 다른 방법을 포함한다.

참고

이 성공기준은 사용자 인터페이스 구성요소가 아닌 글쓰기(writing) 내에서의 섹션을 다룬다. 사용자 인터페이스 구성요소는 성공 기준 4.1.2를 참고하라.

요약

목표
사용자는 콘텐츠가 섹션으로 구성되는 방식을 이해한다.
할 일
콘텐츠가 섹션으로 구성되어 있는 경우 섹션 제목을 제공하라.
중요성
특히 인지 장애나 시각 장애가 있는 사람들은 스스로 방향을 잡을 수 있다.

의도

이 성공 기준의 목적은 페이지가 섹션으로 구성될 때 웹 페이지 섹션에 대한 제목을 제공하는 것이다. 예를 들어 긴 문서는 다양한 장으로 나뉘는 경우가 많고, 장에는 하위 주제가 있는 등의 경우가 있다. 이러한 섹션이 있는 경우 해당 섹션을 소개하는 제목이 있어야 한다. 이는 콘텐츠의 구성을 명확하게 나타내고, 콘텐츠 내 탐색을 용이하게 하며, 콘텐츠 이해를 돕는 심리적 "핸들"을 제공한다. 다른 페이지 요소는 표시를 개선하기 위해 제목을 보완할 수 있지만(예: 가로 규칙 및 상자) 시각적 표시는 문서 섹션을 식별하는 데 충분하지 않다.

이 조항은 모든 유형의 콘텐츠에 적용될 수 없고 제목 삽입이 항상 가능하지 않을 수 있으므로 AAA 수준에 포함된다. 예를 들어, 기존 문서를 웹에 게시할 때 저작자가 원본 문서에 포함하지 않은 제목은 삽입할 수 없다. 또는 긴 편지는 종종 다양한 주제를 다루게 되지만, 편지에 제목을 넣는 것은 매우 이상할 것이다. 그러나 문서를 제목이 있는 섹션으로 나눌 수 있으면 이해와 탐색이 모두 쉬워진다.

이점

  • 시각 장애인은 웹 페이지의 한 섹션에서 다른 섹션으로 이동했을 떄 알 수 있으며 각 섹션의 목적도 알 수 있다.
  • 일부 학습 장애가 있는 사용자는 제목을 사용하여 페이지 콘텐츠의 전체 구성을 더 쉽게 이해할 수 있다.
  • 키보드로 콘텐츠를 탐색하는 사람들은 제목에서 제목으로 초점을 이동할 수 있어 관심 있는 콘텐츠를 빠르게 찾을 수 있다.
  • 페이지의 일부 콘텐츠가 업데이트되는 페이지에서는 제목을 사용하여 업데이트된 콘텐츠에 빠르게 액세스할 수 있다.

예제

  • 메뉴에는 다양한 코스에 대한 다양한 섹션이 포함되어 있다. 각 섹션에는 애피타이저, 샐러드, 수프, 앙트레, 디저트라는 제목이 있다.
  • 웹 애플리케이션에는 관련 설정 그룹으로 구분된 설정 페이지가 포함되어 있다. 각 섹션에는 설정 분류를 설명하는 제목이 포함되어 있다.

기법

이 섹션에서 번호가 매겨진 각 항목은 WCAG 실무 그룹이 이 성공 기준을 충족하기에 충분하다고 간주하는 기법 또는 기법의 조합을 나타낸다. 그러나 이러한 특정 기법을 사용할 필요는 없다. 다른 기법 사용에 대한 자세한 내용은 WCAG 성공 기준에 대한 기법 이해, 특히 "기타 기법" 섹션을 참고하라.

충분 기법

  • G141: Organizing a page using headings
  • H69: Providing heading elements at the beginning of each section of content

Key Terms
section

a self-contained portion of written content that deals with one or more related topics or thoughts

참고

A section may consist of one or more paragraphs and include graphics, tables, lists and sub-sections.

user interface component

a part of the content that is perceived by users as a single control for a distinct function

참고

Multiple user interface components may be implemented as a single programmatic element. "Components" here is not tied to programming techniques, but rather to what the user perceives as separate controls.

참고

User interface components include form elements and links as well as components generated by scripts.

참고

What is meant by "component" or "user interface component" here is also sometimes called "user interface element".

An applet has a "control" that can be used to move through content by line or page or random access. Since each of these would need to have a name and be settable independently, they would each be a "user interface component."
