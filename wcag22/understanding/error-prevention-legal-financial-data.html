성공기준 이해 3.3.4:오류 예방 (법률, 금융, 데이터) (Level AA)

성공 기준 (SC)

사용자에 대한 법률 이행 또는 금융 거래가 발생하거나, 데이터 스토리지 시스템에서 사용자가 제어 가능한 데이터를 수정 또는 삭제하거나, 사용자의 시험(test) 응답을 제출하는 웹 페이지의 경우, 적어도 다음 중 하나를 준수해야 한다.

되돌릴 수 있는
제출 내역을 되돌릴 수 있어야 한다.
점검된
사용자가 입력한 데이터는 입력 오류를 점검하고 사용자에게 오류를 수정할 수 있는 기회를 제공해야 한다.
확인된
제출을 완료하기 전에 정보를 검토, 확인, 수정할 수 있는 매커니즘을 제공해야 한다.

요약

목표
사용자가 중요한 정보를 잘못 제출하는 것을 방지할 수 있다.
할 일
사용자가 중요한 제출물을 확인, 수정 또는 취소할 수 있는 방법을 제공한다.
중요성
장애가 있는 사람들은 실수를 저지르거나 이를 알아차리지 못할 가능성이 더 높다.

의도

이 성공 기준의 목적은 장애가 있는 사용자가 되돌릴 수 없는 작업을 수행할 때 실수로 인해 발생하는 심각한 결과를 방지하도록 돕는 것이다. 예를 들어, 환불되지 않는 항공권을 구매하거나 중개 계좌에 주식 구매 주문을 제출하는 것은 심각한 결과를 초래하는 금융 거래이다. 사용자가 항공 여행 날짜에서 실수할 경우, 교환할 수 없는 잘못된 날짜의 항공권이 나올 수 있다. 사용자가 구매할 주식 수량을 잘못 입력하면 의도한 것보다 더 많은 주식을 구매하게 될 수 있다. 이러한 유형의 실수는 모두 즉시 발생하고 나중에 변경할 수 없는 거래와 관련되어 있으며 비용이 매우 많이 들 수 있다. 마찬가지로, 여행 서비스 웹사이트의 전체 여행 프로필과 같이 나중에 접근해야 하는 데이터베이스에 저장된 데이터를 사용자가 실수로 수정하거나 삭제하는 경우 이는 복구할 수 없는 오류가 될 수 있다. '사용자가 제어할 수 있는' 데이터의 수정 또는 삭제를 언급할 때, 이는 파일이나 기록 삭제와 같은 데이터의 대량 손실을 방지하기 위한 것이다. 각 저장 명령이나 문서, 기록 또는 기타 데이터의 간단한 생성 또는 편집에 대한 확인을 요구하려는 의도는 아니다.

장애가 있는 사용자는 실수할 가능성이 더 높다. 읽기 장애가 있는 사람은 숫자와 문자를 바꿀 수 있고, 운동 장애가 있는 사람은 실수로 키를 누를 수 있다. 작업을 되돌릴 수 있는 기능을 제공하면 사용자는 심각한 결과를 초래할 수 있는 실수를 수정할 수 있다. 정보를 검토하고 수정할 수 있는 기능을 제공하면 사용자는 심각한 결과를 초래하는 행동을 취하기 전에 실수를 발견할 수 있다.

사용자가 제어 가능한 데이터는 사용자가 의도적인 작업을 통해 변경 및/또는 삭제할 수 있는, 사용자가 볼 수 있는 데이터이다. 사용자가 이러한 데이터를 제어하는 ​​예로는 사용자 계정의 전화번호와 주소를 업데이트하거나 웹사이트에서 과거 청구서 기록을 삭제하는 것 등이 있다. 사용자가 직접 보거나 상호작용할 수 없는 인터넷 로그, 검색 엔진 모니터링 데이터 등은 참조하지 않는다.

이점

  • 실수로 인한 심각한 결과를 방지하기 위한 안전 장치를 제공하면 실수할 가능성이 높은 모든 장애가 있는 사용자에게 도움이 된다.

예제

주문 확인
웹 소매업체는 고객에게 온라인 쇼핑을 제공한다. 주문이 제출되면 사용자가 주문 내용이 올바른지 확인할 수 있도록 주문한 품목, 주문한 품목별 수량, 배송지 주소, 결제 방법 등의 주문 정보가 표시된다. 사용자는 주문을 확인하거나 변경할 수 있다.
주식 매매
금융 서비스 웹 사이트에서는 사용자가 온라인으로 주식을 사고 팔 수 있다. 사용자가 주식 매수 또는 매도 주문을 제출하면 시스템은 시장이 열려 있는지 여부를 확인한다. 영업 시간 이후인 경우 해당 거래가 영업 시간 외 거래가 될 것이라는 경고를 사용자에게 알리고, 정규 시장 시간 이외의 거래에 따른 위험에 대해 설명하며, 주문을 취소하거나 확인할 수 있는 기회를 제공한다.

기법

이 섹션에서 번호가 매겨진 각 항목은 WCAG 실무 그룹이 이 성공 기준을 충족하기에 충분하다고 간주하는 기법 또는 기법의 조합을 나타낸다. 그러나 이러한 특정 기법을 사용할 필요는 없다. 다른 기법 사용에 대한 자세한 내용은 WCAG 성공 기준에 대한 기법 이해, 특히 "기타 기법" 섹션을 참고하라.

충분 기법

아래에서 귀하의 콘텐츠와 일치하는 상황을 선택하라. 각 상황에는 해당 상황에 충분하다고 알려져 있고 문서화된 기법 또는 기법 조합이 포함된다.

상황 A: If an application causes a legal transaction to occur, such as making a purchase or submitting an income tax return:

  • G164: Providing a stated time within which an online request (or transaction) may be amended or canceled by the user after making the request
  • G98: Providing the ability for the user to review and correct answers before submitting
  • G155: Providing a checkbox in addition to a submit button

상황 B: If an action causes information to be deleted:

  • G99: Providing the ability to recover deleted information
  • G168: Requesting confirmation to continue with selected action
  • G155: Providing a checkbox in addition to a submit button

상황 C: If the Web page includes a testing application:

  • G98: Providing the ability for the user to review and correct answers before submitting
  • G168: Requesting confirmation to continue with selected action

조언 기법

준수를 위해 필수는 아니지만 콘텐츠에 더 쉽게 접근할 수 있도록 다음과 같은 추가 기법을 고려해야 한다. 모든 기법을 사용할 수 없거나 모든 상황에서 효과적인 것은 아니다.

  • SCR18: Providing client-side validation and alert
  • G199: Providing success feedback when data is submitted successfully
